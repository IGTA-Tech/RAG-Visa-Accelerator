# NOTEBOOKLM BUILD GUIDE
## Step-by-Step Instructions for Creating Each Training Module

---

## HOW TO USE THIS GUIDE

For each module, this guide tells you:
1. **What notebook to create** (name and purpose)
2. **What sources to add** (documents from RAG repo)
3. **What questions to prime** (for the AI to learn)
4. **What audio overview to generate** (for podcast-style learning)
5. **How to test** (validation questions)

---

## NOTEBOOKLM BEST PRACTICES

### Source Organization
- Maximum 50 sources per notebook
- Mix document types (MD, YAML, PDF, text)
- Include Q&A content for better responses
- Add the canonical-source-of-truth.yaml to EVERY notebook

### Naming Convention
```
IGTA Training - Module [X] - [Module Name]
```

### Audio Overview Tips
- Generate after all sources are added
- Review for accuracy
- Regenerate if major errors
- Save audio for offline training
